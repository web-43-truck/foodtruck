Joe Slocum

Demographic

Joe Slocum is a foodie and full stack developer who maintains an independent non-monetized food truck website that he maintains in his spare time

Tech Bio

Joe programs on a Linux based laptop that was custom-built with the latest chip set. He owns several other laptops that run various types of os that he uses to preview his work before deployment

usage Scenario

Typical usage may include correcting errors in code, responding to front-end user needs as wells as research and recruiting to keep his site current

Attitudes and Attributes

  • a site should be easily accessible and aesthetically pleasing
  • A sites code should be clean, clear, concise, and accessible for updates or changes
  • Front-end users should be able to find information on a site quickly and easily
  • Commercial sites such as foodtruchhunter.com should not contain 3rd party adds.

Frustrations and Limitations

  1. Pages that are so static they become boring
  2. sites that are poorly organized or difficult to navigate
  3. sites that are bereft of pertinent information
  4. sites that are too busy, makes poor coloration choices or are otherwise aesthetically unpleasing

Goals

  1. to maintain a successful site where consumers can find up-to-date information on local food trucks.
  2. To respond quickly, efficiently and effectively to front-end user issues
  3. To provide a site that is dynamic, interactive and aesthetically pleasing to the front-end user

Interaction Flow

Precondition: The site is set to its default facade but needs adjusted for an upcoming holiday

  1. Joe signs on to his site
  2. The site confirms Joe's identity and role granting him the necessary permission to make changes to the site as a whole
  3. Joe starts his preferred IDE
  4. The idea connects to the server and gitHub upon opening and returns Joe to the Last project he worked on
  5. Joe heads pulls from git the code for the site
  6. GitHub retrieves the repository and places it in Joe's RAM
  7. Joe navigates the menu and opens food-truckHunter .com's css, js and index.html files
  8. Joe's IDE opens the files
  9. using his preferred ftp method Joe uploads several image of an upcoming holiday into the images directory
  10. The IDE and terminal both confirm the uploads
  11. Joe opens a new document and builds a template for the upcoming holiday that sets a theme using colors and font commonly associated with holiday
  12. the IDE process the new code and previews it in a browser
  13. Joe goes to his index.html file and changes the script to engage the new theme
  14. The preview shows a few errors in sizing and spacing of some of the images that are overlaying important interaction points.
  15. Joe returns to the code adjusting the errors he noted
  16. The browser preview auto updates showing continued spacing issues with the key image for the theme.
  17. Joe adds a margin to the image
  18. the browser auto updates showing the corrected image as working the way Joe would like
  19. Joe sees a few more issues and adjusts them continuing to commit after each change
  20. the browser continues to auto update and git continues to confirm the commits/pushes
  21. Joe finalizes the ui/ux commits, pushes the holiday themed template
  22. the preview satisfactory and exceeds Joe's original idea for the theme
  23. Joe deploys the new theme
  24. The server caches the new theme and will deploy with each new opening by a front-end user.
  25. Joe sign off, uses his app to find a good coffee truck, travels to the truck and enjoys a cup of Joe.

Post-condition: The sites facade uses the new template to celebrate the upcoming holiday
